## 🔐 الأمان:

### ⚠️ تذكر:
1. **غيّر كلمة المرور الافتراضية**
   - في ملف `admin/dashboard.html`
   - ابحث عن: `ADMIN_PASSWORD: 'admin123'`
   - غيّرها إلى كلمة قوية

2. **احفظ GitHub Token في مكان آمن**
   - لا تشاركه مع أحد
   - استخدم Environment Variables فقط

3. **استخدم HTTPS فقط**
   - تأكد من أن الموقع يستخدم HTTPS

## 🔄 سير العمل:

### إضافة منتج:
1. ادخل لوحة التحكم
2. اضغط "إضافة منتج جديد"
3. ملأ البيانات
4. اضغط "إضافة"
5. البيانات تُحفظ على GitHub تلقائياً

### تعديل منتج:
1. ابحث عن المنتج
2. اضغط "تعديل"
3. غيّر البيانات
4. اضغط "تحديث"
5. التغييرات تُحفظ على GitHub

### حذف منتج:
1. ابحث عن المنتج
2. اضغط "حذف"
3. أكد الحذف
4. المنتج يُحذف من GitHub

## 🎯 الخطوات التالية:

### 1. دفع الملفات إلى GitHub:
```bash
git add .
git commit -m "Add admin dashboard with GitHub API integration"
git push origin main
```

### 2. Vercel سينشر تلقائياً:
- Vercel يراقب GitHub
- عند كل push، يتم النشر تلقائياً

### 3. تحقق من النشر:
- اذهب إلى: https://vercel.com/dashboard
- تحقق من أن النشر نجح

### 4. جرّب لوحة التحكم:
- اذهب إلى: https://tiqtaqo-frontend-o2fa.vercel.app/admin/dashboard.html
- ادخل البيانات
- أضف منتج تجريبي

## 🐛 استكشاف الأخطاء:

### المشكلة: "Failed to fetch file"
**الحل:**
- تأكد من GitHub Token صحيح
- تأكد من Repository name صحيح
- تأكد من ملف `data/products.json` موجود

### المشكلة: "Unauthorized"
**الحل:**
- تأكد من GitHub Token صحيح
- أنشئ Token جديد إذا لزم الأمر

### المشكلة: "كلمة المرور غير صحيحة"
**الحل:**
- كلمة المرور الافتراضية: `admin123`
- تأكد من كتابتها بشكل صحيح
